## Auth Bug: PKCE Code Verifier Not Found (Root Cause Found)

SDK source: `@supabase/supabase-js/dist/umd/supabase.js`
Error message: "code verifier not found in storage. This can happen if the auth flow
was initiated in a different browser or device"

With PKCE flow (flowType: 'pkce'), the SDK stores the code_verifier in localStorage
using a key tied to the login page's URL. When Google redirects back to the callback
page, the callback page can't access the verifier → PKCE exchange fails → "Session
exchange failed".

FIXED: Switched to implicit flow (flowType: 'implicit'). Token comes back in URL fragment.
Callback parses fragment directly, no PKCE verifier needed.
