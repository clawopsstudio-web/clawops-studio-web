3. **Privacy anxiety** — enterprise data leaking to third-party AI APIs
4. **Siloed AI** — copilots help ONE person, not the organization
5. **No production path** — tools you demo vs. tools you run in production
6. **Fragmented channels** — agents live in one app, not everywhere
7. **No mobile command** — you're chained to your desktop to manage AI

- **Topic Routing** (v3.7) — one group, multiple agents, each with different models + skills. No skill bloat.

- **Model-agnostic** — DeepSeek for cheap tasks, GPT-4 for reasoning, Gemma for local. You pick per-task.
- **The economics** — $49/mo flat. Compare to $3,000+ for a human employee doing the same work.

## 3D Scroll Animation Fix

The current implementation in HeroNew.tsx uses `useScroll` + `useTransform` from Framer Motion. Pulkit says he "can't see it."

**Possible issues:**
1. The scroll progress only tracks the full page scroll — `useScroll()` defaults to `window` scroll
2. The animations may not trigger on mobile (touch scroll is different)
3. The `FloatingShapes` are `absolute` inside an `overflow-hidden` parent — they render but may not be visually prominent enough
4. The `PerspectiveGrid` uses CSS `perspective(400px) rotateX(65deg)` — this should be visible

**Fix plan:**
- Make the 3D shapes MORE visible (increase opacity, size, color saturation)
- Add CSS `will-change: transform` for GPU acceleration
- Ensure the grid is clearly visible (increase line opacity)
- Add a subtle "scan line" or "horizon glow" at the bottom to anchor the 3D feel
- Make the floating cards more prominent with stronger shadows + borders
- Add a gentle parallax on the hero text itself (text moves at different speed than background)
